# WB Daily Operations Review

WB 每日运营复盘与广告策略决策系统。第一阶段聚焦 **Excel 模板解析 + 历史数据保存**：每天维护一个 Excel 文件，每个 SKU 是一个独立 sheet，每天新增一行数据，导入后系统按“日期 + SKU”覆盖保存到浏览器 IndexedDB。

## 当前阶段能力

- 支持上传 `.xlsx/.xls` Excel 文件。
- 自动读取形如 `ES032BK`、`ES030BK`、`ES068BK`、`ES113BO`、`ES666G`、`ES920E`、`ES823E` 的 SKU sheet。
- 自动跳过 `wb利润定价表`、`ozon利润定价表`、`Sheet10`、汇总、说明、模板等辅助 sheet。
- 每个 SKU sheet 解析为统一每日运营数据结构。
- 以 `日期 + SKU` 作为唯一键写入 IndexedDB，重复导入同一天同一个 SKU 会覆盖更新，不会重复新增。
- 提供导入数据记录：已导入日期、SKU 数量、总订单、总广告费、总利润。
- 支持按日期和 SKU 筛选历史数据。
- 支持导出备份 JSON 与导入备份 JSON，降低浏览器数据丢失风险。

## 快速开始

```bash
npm run dev
```

浏览器打开 <http://localhost:5173> 后上传 Excel 文件。当前实现为零 npm 依赖；Excel 解析会在浏览器端动态加载 SheetJS CDN。


## GitHub Pages 部署

本项目已配置 GitHub Actions 自动部署到 GitHub Pages：

- 推送到 `main` 或 `master` 分支时会自动运行 `.github/workflows/deploy.yml`。
- 也可以在 GitHub 仓库的 **Actions** 页面手动运行 `Deploy to GitHub Pages` 工作流。
- 部署完成后，在工作流运行详情的 `deploy` job、仓库 **Settings → Pages**，或 GitHub Actions 生成的 `github-pages` 环境中查看访问链接。
- Pages 站点发布的是 `dist/` 静态产物；构建会保留 `index.html`、`src/` 和 `sample-data/`，页面仍在浏览器端使用 IndexedDB 保存历史数据和动作记录，并通过 SheetJS CDN 解析 Excel。

## SKU sheet 识别规则

有效 SKU sheet 名称需要匹配类似 `ES032BK` 的格式：英文字母开头 + 至少 3 位数字 + 可选字母/数字后缀。

以下 sheet 会被视为辅助 sheet 并跳过：

- 包含 `利润定价表`、`wb利润`、`ozon` 的 sheet。
- `Sheet10`、`Sheet1` 等默认辅助 sheet。
- 包含 `汇总`、`说明`、`模板` 的 sheet。

## 历史数据与备份

- 数据优先保存在浏览器 IndexedDB，数据库名为 `wb-daily-ops-review`。
- 数据主键为 `日期__SKU`，例如 `2026-06-18__ES032BK`。
- 点击“导出备份 JSON”可下载当前全部历史记录。
- 点击“导入备份 JSON”会用备份内容替换当前浏览器中的历史记录。

## 第二阶段：每日动作记录

现在可以在页面中维护结构化的每日运营动作记录。动作记录与导入数据使用同一个唯一键：`日期__SKU`。

### 动作记录字段

- 广告类型：`CPC`、`CPM搜索`、`CPM推荐`、`统一出价`、`手动出价`。
- 出价与预算：搜索出价、推荐位出价、每日广告预算。
- 预算动作：加大预算、降低预算、保持预算、暂停广告。
- 价格动作：涨价、降价、保持价格、参加活动。
- 主图动作：更换主图、未更换。
- 关键词动作：调整标题、调整关键词、未调整。
- 库存动作：补货、库存不足、库存正常。
- 备注：记录当天额外判断、执行原因或复盘结论。

### 保存、编辑与删除

- 在“每日动作记录”模块选择日期和 SKU 后填写表单，点击保存即可写入 IndexedDB。
- 同一天同一个 SKU 再次保存会覆盖旧动作，适合日内修正。
- 点击历史明细中的“查看详情”会把对应日期和 SKU 的动作加载到表单，可继续编辑或删除。
- “SKU 详情”会同时展示 Excel 导入的“运营动作原文”和系统结构化动作。
- 如果 Excel 的 B 列或对应表头列已有运营动作原文，系统会先导入保存；后续复盘和决策以结构化动作表单为准。
- 备份 JSON 会同时包含历史每日数据和动作记录。

## 第三阶段：动作效果分析和明日策略建议

系统会结合历史每日数据和结构化动作记录，对每个 SKU 自动生成动作效果判断、风险提示和明日策略建议。

### 对比口径

- 今天 vs 昨天。
- 近 3 天平均。
- 近 7 天平均。
- 动作前 3 天 vs 动作后 3 天。

### 重点指标

分析会覆盖订单量、销售额、广告费、广告占比、利润、利润率、CTR、CVR、ACOS、ROI、库存、价格、评论数。

### 规则示例

- 提高出价后广告费上升、订单没增长、利润下降：判断动作效果差，建议控制广告花费。
- 降低预算后订单稳定、利润提升：判断控费有效，建议保持当前策略并继续观察。
- 提高推荐位预算后曝光增加但点击率下降：判断推荐流量质量差，建议降低推荐位预算。
- 搜索广告花费高但转化率低：建议降低搜索出价或优化关键词。
- 有曝光没点击：建议优化主图、标题或价格。
- 有点击没转化：建议优化价格、评价、详情页，可降价或参加活动。
- ROI 和利润持续为正且库存充足：建议适当加大预算。
- 库存不足：不建议继续加大广告，优先补货。
- 利润连续为负：建议降低广告预算或暂停广告。

### 页面模块

- 明日策略建议看板：展示每个 SKU 的主要建议和原因。
- 动作效果分析卡片：展示动作有效性和关键指标对比。
- SKU 风险提示：集中展示库存不足、无点击、无转化、连续亏损等风险。
- 建议历史：跟随日期和 SKU 筛选，查看建议类型、优先级和原因。
